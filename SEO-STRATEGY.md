# 3. Technical SEO Implementation

## 3.1 Fix Broken Sitemap (CRITICAL -- Do First)

The current sitemap-index.xml returns a 404 error. This must be fixed immediately.

**Implementation in Next.js:**

Create `app/sitemap.ts` (or `app/sitemap-index.xml/route.ts`):

```typescript
// app/sitemap.ts
import { MetadataRoute } from 'next'

export default function sitemap(): MetadataRoute.Sitemap {
  const baseUrl = 'https://getswiftbill.app'

  const routes = [
    // English pages
    { url: `${baseUrl}/en`, lastModified: new Date(), changeFrequency: 'weekly', priority: 1.0 },
    { url: `${baseUrl}/en/templates`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/en/zatca-invoicing`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/en/uae-invoicing`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/en/privacy`, lastModified: new Date(), changeFrequency: 'yearly', priority: 0.3 },
    { url: `${baseUrl}/en/terms`, lastModified: new Date(), changeFrequency: 'yearly', priority: 0.3 },
    // Arabic pages
    { url: `${baseUrl}/ar`, lastModified: new Date(), changeFrequency: 'weekly', priority: 1.0 },
    { url: `${baseUrl}/ar/templates`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/ar/zatca-invoicing`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/ar/uae-invoicing`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    // Blog posts (add as created)
  ]

  return routes as MetadataRoute.Sitemap
}
```

Update `robots.txt` to point to the correct sitemap URL:
```
User-agent: *
Allow: /

Sitemap: https://getswiftbill.app/sitemap.xml
```

## 3.3 Hreflang Tags (Verify & Enhance)

Current implementation has basic hreflang. Ensure EVERY page has complete hreflang tags:

```html
<!-- On /en pages -->
<link rel="alternate" hreflang="en" href="https://getswiftbill.app/en" />
<link rel="alternate" hreflang="ar" href="https://getswiftbill.app/ar" />
<link rel="alternate" hreflang="x-default" href="https://getswiftbill.app/en" />

<!-- On /en/blog/some-post pages -->
<link rel="alternate" hreflang="en" href="https://getswiftbill.app/en/blog/some-post" />
<link rel="alternate" hreflang="ar" href="https://getswiftbill.app/ar/blog/some-post" />
<link rel="alternate" hreflang="x-default" href="https://getswiftbill.app/en/blog/some-post" />
```

**Critical rules:**
- Every page must have hreflang tags pointing to ALL language versions
- Always include `x-default` pointing to the English version
- For region-specific content, consider `hreflang="en-SA"` and `hreflang="ar-SA"` for Saudi-specific pages
- Hreflang must be bidirectional (if /en points to /ar, then /ar must point to /en)
